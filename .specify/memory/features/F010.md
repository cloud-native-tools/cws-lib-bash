# Feature Detail: Cloud Infrastructure

**Feature ID**: F010
**Name**: Cloud Infrastructure
**Description**: Utilities for cloud providers (Alibaba Cloud ECS) and Infrastructure-as-Code (Terraform).
**Status**: Implemented
**Created**: 2026-06-15
**Last Updated**: 2026-06-15

## Overview

Provides shell functions for querying Alibaba Cloud ECS instance metadata (VPC, region, IP addresses, instance type) and managing Terraform workflows (plan, apply, destroy, format, file cleanup). Enables cloud infrastructure operations directly from the shell.

## Latest Review

Initial feature definition extracted from existing codebase scan. Covers `aliyun.sh` (17 functions) and `terraform.sh` (20 functions).

## Key Changes

1. ECS metadata query functions (VPC, vswitch, region, zone, instance type, IPs)
2. Terraform lifecycle management (plan, apply, destroy)
3. Terraform file utilities (format, cleanup unused/plan files)
4. Terraform data reading from YAML and JSON sources

## Implementation Notes

- `aliyun.sh` relies on Alibaba Cloud ECS metadata service endpoint availability.
- `terraform.sh` wraps the `terraform` CLI; requires `terraform` binary in PATH.
- All functions use parameterized inputs; no hardcoded cloud account or project references.

## Future Evolution Suggestions

- Add support for additional cloud provider metadata APIs (AWS IMDSv2, GCP metadata server).
- Terraform state management and workspace switching utilities.
- Cloud resource tagging and cost-reporting helpers.
