# Project Instructions for AI Agents

## Project Overview
CWS-Lib-Bash is a general-purpose Bash utility library for cloud-native environment operations, system management, and development workflows. The library is organized by technology domain in `scripts/*.sh` with foundational capabilities in `profile.d/*.sh`. It is loaded into the current shell via `source ./bin/cws_bash_env` or executed function-by-function via `./bin/cws_bash_run <function_name> [args...]`.

**Core design principle**: Maximum universality is the #1 priority. All functions MUST be general-purpose with zero project-specific or customization logic. Apply this test: "Would a developer on a completely unrelated project find this function useful as-is?" If not, the code does not belong here.

  - `expect/`: Expect automation interaction scripts

## Runtime Loading Model (Critical)
- `bin/cws_bash_env` sources `profile.d/*.sh` then `scripts/*.sh` in order — code executes at load time.
- Load-time code MUST be silent and lightweight: no unconditional stdout/stderr, no expensive external commands, no side effects.
- Use `return ${RETURN_FAILURE:-1}` in functions, NEVER `exit` (would terminate the caller's shell).
- Check dependencies with `have <cmd>` before using external commands.
- Follow: `snake_case` naming, `local` variables, `${var}` brace quoting, `${var:-default}` defaults.

## Build/Test/Validation Commands
- Load environment: `source ./bin/cws_bash_env`
- Run a function: `./bin/cws_bash_run <function_name> [args...]`
- Run all tests: `./bin/cws_bash_test`
- Run single test: `./bin/cws_bash_test test_env`
- Lint all scripts: `find "$PWD" -type f -name "*.sh" | xargs shellcheck -x`
